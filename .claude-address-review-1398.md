#1398 [refactor] Remove 3 identical _find_pr_for_issue delegation methods — patch _review_utils directly

## Motivation

`_find_pr_for_issue` exists as an identical 5-line method in 3 classes (`PRReviewer`, `AddressReviewer`, `CIDriver`). Each is a trivial delegation to `_review_utils.find_pr_for_issue` with slight variations on the `extra_strategies` parameter. These exist as test seams so `patch.object(cls, "_find_pr_for_issue")` works, but the test patches could target `_review_utils.find_pr_for_issue` directly.

## Proposed Solution

Remove the `_find_pr_for_issue` methods from all 3 classes. Update tests to patch `_review_utils.find_pr_for_issue` (or `hephaestus.automation._review_utils.find_pr_for_issue`). This eliminates 3 thin wrappers and makes the call path clearer.

Alternatively, if backward-compatible test seams are preferred, keep the methods but extract them into a mixin or base class.

## Impact

- ~30 lines removed across 3 files
- Test patches updated (mechanical find-and-replace)
- Low risk

## Files affected

- `hephaestus/automation/pr_reviewer.py` (remove `_find_pr_for_issue`)
- `hephaestus/automation/address_review.py` (remove `_find_pr_for_issue`)
- `hephaestus/automation/ci_driver.py` (remove `_find_pr_for_issue`)
